Project Explanation

This system enhances safety in mining operations by monitoring lift mechanisms wirelessly. It integrates multiple sensors to detect overloads, misalignments, and emergency stops, ensuring real-time responses to prevent accidents. The design includes wireless communication for remote monitoring, allowing operators to oversee multiple lifts from a central station.

Microcontroller (MCU) Details

The ESP32 microcontroller was chosen for its dual-core processing, built-in Wi-Fi and Bluetooth capabilities, and extensive GPIO pins. It handles RTOS (FreeRTOS) for multitasking, managing sensor interrupts, data logging, and wireless transmission efficiently. Key features utilized: ADC for analog sensor readings, SPI/I2C for peripheral communication, and low-power modes for battery-operated remote units.

Components Used

  • ESP32 DevKitC board as the core MCU
  • Hall-effect sensors for position detection
  • Load cells (HX711 amplifier) for weight monitoring
  • Accelerometers (MPU-6050) for vibration and tilt sensing
  • Relay modules for emergency stop controls
  • LoRa modules (SX1276) for long-range wireless communication in mine environments
  • Power management: LiPo batteries with TP4056 chargers and buck converters
  • Display: OLED SSD1306 for local status indicators
  • Enclosures: IP67-rated for dust and water resistance

Prospects and Future Enhancements

This project has strong prospects in industrial safety applications, potentially expanding to other heavy machinery like cranes or elevators. Future enhancements could include AI-based predictive maintenance using machine learning on sensor data, integration with cloud platforms for analytics, and certification for international mining standards. It could reduce downtime by 30% and improve safety compliance, opening doors to partnerships with mining companies.

ESP32 Battery Capacity Tester

Controller: ESP32 | Firmware: Arduino (PlatformIO)

Constant-current electronic load with calibrated ADC, DAC-controlled MOSFET, touch UI.

const float ADC_LUT[4096] = { }; // generate yours from https://github.com/e-tinkers/esp32-adc-calibrate

// defining pins
const int led = 2;
const int touch = 4;
const int dac1 = 25;
const int dac2 = 26;
const int adc = 35;

const int touchThreshold = 30;
const int minBatteryVoltage = 2800; // voltage * 100

int adcValue = 0;

bool isTouched() {
  if (touchRead(touch) < touchThreshold)
  {
    delay(500);
    if (touchRead(touch) < touchThreshold)
    {
      return true;
    }
    else {
      return false;
    }
  }
  else {
    return false;
  }
}

bool needsReset() {
  // checking if pin is kept touched for 3 sec
  if (isTouched()){
    for(unsigned int i=0;i<3;i++){
      if (!isTouched()){
        return false;
      }
      delay(500);
    }
    return true;
  }
  else {
    return false;
  }
}

void blinkLed() {
  while (true) {
    // blink twice
    for(unsigned int i=0;i<2;i++){
      digitalWrite(led, HIGH);
      delay(100);
      digitalWrite(led, LOW);
      delay(100);
    }
    if (needsReset()){
      // resetting, blink ten times
      Serial.println("Resetting in 5 sec ...");
      for(unsigned int i=0;i<10;i++){
        digitalWrite(led, HIGH);
        delay(100);
        digitalWrite(led, LOW);
        delay(100);
      }
      // wait for 5 sec then blink for 1 sec
      delay(5000);
      digitalWrite(led, HIGH);
      delay(1000);
      digitalWrite(led, LOW);
      break;
    }
    delay(3000);
  }
}

int batteryVoltage() {
  int calibratedReading = 0;
  // Taking 10 adc samples for stable reading
  for(unsigned int i=0;i<10;i++){
    adcValue = analogRead(adc);
    calibratedReading = calibratedReading + (int)ADC_LUT[adcValue];
    delay(10);
  }
  int avgreading = calibratedReading/10;

  // Adc value to voltage mapping multiplied by voltage divider factor
  return (map(avgreading, 0,4095,0,3300)) * 2;
}

void testCompleted(){
  // Job is done
  // set dac to zero to stop drain and watch
  dacWrite(dac1, 0);
  dacWrite(dac2, 0);
  Serial.println("Test Completed");
  blinkLed();
}

void setup() 
{
  Serial.begin(115200);
  pinMode (led, OUTPUT);
  dacWrite(dac1, 0);
  dacWrite(dac2, 0);
  digitalWrite(led, LOW);
}
void loop() 
{
  // Wait for touch
  Serial.println("Waiting for touch...");
  while (!isTouched()) {
    delay(1000);
  }
  digitalWrite(led, HIGH);

  if (batteryVoltage() <= minBatteryVoltage){
    testCompleted();
  }
  else {
    // start draining Battery and Watch
    dacWrite(dac1, 77); // 1v to op-amp
    dacWrite(dac2, 116); // 1.5v to analog watch
    Serial.println("Test Started");
    int voltage = 0;
    while (true){
      // Wait for Battery to drain checking every 1 sec.
      voltage = batteryVoltage();
      if (voltage > minBatteryVoltage) {
        Serial.print("Battery Voltage: ");
        Serial.println((float)voltage/1000);
        delay(1000);
      }
      else {
        break;
      }
    }
    testCompleted();
  }
}

Project Explanation

A constant-current electronic load discharges a cell while the ESP32 logs voltage via a calibrated ADC. DAC1 biases an LM358 that drives a logic-level N-MOSFET; current is sensed across a 1 Ω shunt and kept roughly constant by the op-amp loop. A resistor divider (R1=10 kΩ, R2=10 kΩ) scales the battery voltage to the ADC input. Touch input is used as a single-button UI to start/stop and request reset.

Microcontroller (MCU) Details

ESP32 pinout: ADC on GPIO35 (input-only), DAC1 on GPIO25 to the LM358 non-inverting input, DAC2 on GPIO26 to an external analog reference/watch, capacitive touch on GPIO4, and LED on GPIO2. Raw ADC readings are linearized with a per-code LUT generated by the esp32-adc-calibrate tool to remove slope/offset non-linearities. The firmware samples 10× with short delays, averages, maps to millivolts, and applies the divider ratio. For larger tests, FreeRTOS tasks can separate UI, sampling, and logging.

Electronics Notes

  • LM358 powered at 5 V guarantees gate drive margin; choose a MOSFET with low Rds(on) at Vgs≈4.5 V.
  • Place the 1 Ω shunt in the low side; use Kelvin connections to the op-amp sense to minimize error.
  • Add a small RC (e.g., 100 Ω series at the op-amp output and 10–47 nF across sense) to stabilize the loop.
  • Divider R1/R2=10 kΩ keeps source impedance low for ADC; add 100 nF to ground at the ADC node.
  • Thermal: a 1 Ω/5 W shunt and MOSFET need heatsinking; monitor case temperature.

KiCad

Schematic implements ESP32-DevKit, LM358, N-MOSFET, 1 Ω shunt, divider, TVS and reverse-polarity protection. PCB uses star ground at the shunt return, wide copper for load current, and an analog ground island for the ADC node with a single-point tie.

PlatformIO

Environment: platform = espressif32, board = esp32dev, framework = arduino. Use build_flags = -O2 -DCORE_DEBUG_LEVEL=1. Add a task to generate/pack the LUT into a header; CSV logging at 115200 baud for post-processing mAh/Wh.
